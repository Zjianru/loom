# 设计对齐纪要

状态：alignment draft  
定位：本轮对齐纪要，不覆盖现有 target 主链  
更新时间：2026-03-10

---

## 1. 一句话定义
我们不是在卖平台，不是在卖模型，也不是在卖 agent marketplace。  
我们在卖：

**面向个人 AI 终端的协作治理能力层。**

更具体地说：
1. assistant 继续做用户入口
2. 更具体、更专业、更重的工作交给更有能力的 agent
3. 我们只做编排、治理、提醒、记录、验收边界

---

## 2. 产品边界
### 2.1 我们卖什么
我们卖的是：
1. 复杂任务被正式扔出去后，还能持续推进
2. 用户不需要 babysit
3. 主 assistant 在后台任务执行时仍然可用
4. 进展有人主动汇报
5. 过程有人持续记录
6. 写入和高风险动作始终有边界
7. 复杂任务有阶段、有角色、有验收

### 2.2 我们不卖什么
我们不卖：
1. 通用大而全平台
2. 通用低代码 builder
3. 通用模型能力
4. 各领域具体执行能力本身

这些能力应该尽量交给更强的模型和更专业的 agent。

---

## 3. 核心判断
### 3.1 现在缺的不是能力层，而是治理层
很多个人 AI 终端的问题不是：
1. 模型不够强
2. 工具不够多

而是：
1. 没有正式任务对象
2. 没有阶段治理
3. 没有 agent 协作治理
4. 没有过程沉淀
5. 没有异步通知
6. 没有验收和返工机制

### 3.2 目标用户是个人级 AI terminal 用户
我们的目标不是只服务企业 builder 用户。  
我们要服务的是：
1. 正在使用 OpenClaw、类 OpenClaw、Chat-first AI terminal 的个人用户
2. 希望把 assistant 从聊天工具升级成“一人公司入口”的用户

---

## 4. 总体架构
### 4.1 两类资源池
#### `StagePool`
它是什么：
1. 阶段池

它代表什么：
1. 一组可复用的阶段模板
2. 用户可组合的阶段积木
3. 任务推进方式的池化能力

它不是什么：
1. 不是具体某次任务实例
2. 不是 agent roster

#### `AgentPool`
它是什么：
1. agent 池

它代表什么：
1. 一组可实例化的 agent 能力单元
2. 用户可组合的人才池
3. 任务执行角色的池化能力

它不是什么：
1. 不是阶段模板
2. 不是任务状态

### 4.2 三条运行线
#### `Harness`
它是什么：
1. 主运行时

它负责：
1. 任务激活
2. 阶段流转
3. 阶段编排
4. agent 编排
5. 任务推进

#### `watchdog`
它是什么：
1. 通知运行时

它负责：
1. 主动提醒
2. 卡点上报
3. 阶段完成通知
4. 需要用户拍板时的升级通知

#### `approval-gate`
它是什么：
1. 边界运行时

它负责：
1. 写入边界
2. 高风险工具边界
3. 执行授权边界

在没有得到同意的情况下，它永远不能允许写入。

### 4.3 recorder 的位置
`recorder` 先按 first-class agent role 处理，而不是第四个插件。

原因：
1. recorder 更像“正式工作角色”，不是底层拦截 runtime
2. recorder 需要参与阶段、产出文档、进入记忆系统
3. 如果 recorder 也做成插件，运行线会过多，耦合会更高

## 5. 核心对象
### 5.1 `interactionLane`
它是什么：
1. 顶层仲裁结果

它表示：
1. 当前输入留在普通聊天
2. 还是进入受管任务

建议值：
1. `chat`
2. `managed_task_candidate`
3. `managed_task_active`

取舍：
1. 这个变量解决“插件要不要激活”
2. 它不负责描述任务类型

### 5.2 `managedTaskClass`
它是什么：
1. 单个受管任务的协作拓扑和治理强度定义

正式值：
1. `COMPLEX`
2. `HUGE`
3. `MAX`

取舍：
1. 它不再是“从小到大”的一般工作量刻度
2. 它表示协作框架

### 5.3 `managedTaskRef`
它是什么：
1. 单个受管任务的一等对象

它代表什么：
1. start card 的 owner
2. 执行过程的 owner
3. 汇总与关闭的 owner

### 5.4 `activeManagedTaskRef`
它是什么：
1. 当前唯一活跃任务的轻量指针

它代表什么：
1. 当前会话正在跑哪个受管任务

它不代表什么：
1. 不代表整个聊天被冻结

### 5.5 `taskConversationPolicy`
建议默认值：
1. `chat_open`

它表示：
1. active task 存在时，主 assistant 是否继续可用

### 5.6 `StagePackage`
它是什么：
1. 单个阶段模板

它代表什么：
1. 阶段目标
2. 阶段输入
3. 阶段输出
4. 进入条件
5. 退出条件
6. 允许调用的 agent 类型

### 5.7 `PhasePlan`
它是什么：
1. 某个任务当前选用的阶段组合

它代表什么：
1. 阶段顺序
2. 阶段依赖
3. 阶段是否可跳过、重做、并行

### 5.8 `AgentCapabilityProfile`
它是什么：
1. 单个 agent 的能力画像

它代表什么：
1. 擅长什么
2. 用什么模型
3. 能用什么工具
4. 风险等级
5. 成本等级

### 5.9 `AgentBinding`
它是什么：
1. 某个阶段实际绑定到哪些 agent

它代表什么：
1. 当前实际上场的人手
2. 他们各自承担什么角色

### 5.10 `RecorderAgentRef`
它是什么：
1. 当前任务的记录者角色引用

它代表什么：
1. 是否启用 recorder
2. 谁负责过程沉淀
3. 谁负责阶段总结和交付记忆整理

### 5.11 `watchdogNoticeState`
它是什么：
1. watchdog 已经通知到哪一步

它代表什么：
1. 哪些进展已经发过
2. 哪些卡点还没告知
3. 哪些提醒需要去重

### 5.12 `AcceptancePolicy`
它是什么：
1. 任务验收策略

它代表什么：
1. 什么叫完成
2. 谁来验收
3. 最多返工几轮
4. 什么情况下必须升级给用户

## 6. 三种 managed class 的定义
### 6.1 `COMPLEX`
它是什么：
1. `1 + 1` 型受管协作

典型角色：
1. `net`
2. 一个专业 worker
3. 一个 recorder
4. `watchdog`

核心体验：
1. 用户正式把任务委派出去
2. worker 持续处理
3. recorder 沉淀过程
4. watchdog 主动汇报
5. `net` 继续处理别的事

### 6.2 `HUGE`
它是什么：
1. `1 + n` 型受管协作

典型角色：
1. `net`
2. 多个 worker
3. 一个 recorder
4. `watchdog`

核心体验：
1. 多阶段推进
2. 多专业角色参与
3. 用户仍从 `net` 统一入口发起变更、补充、暂停、取消

### 6.3 `MAX`
它是什么：
1. 项目级/工程级分层协作

典型特征：
1. worker 可以继续派生 worker
2. 拓扑层级更深
3. 治理和边界必须更保守

---

## 7. 角色边界
### 7.1 `net`
负责：
1. 用户入口
2. 任务激活
3. 召集计划
4. 接收用户变更
5. 汇总最终结果

不负责：
1. 全程盯执行细节
2. 自己做持续记录
3. 自己做主动通知

### 7.2 worker
负责：
1. 专业执行
2. 阶段目标产出

### 7.3 recorder
负责：
1. 过程记录
2. 经验沉淀
3. 阶段性总结
4. 面向记忆系统的结构化产物

### 7.4 `watchdog`
负责：
1. 主动通知
2. 卡点升级
3. 提醒去重

不负责：
1. 做主决策
2. 替代 recorder

### 7.5 `approval-gate`
负责：
1. 所有写操作和关键高风险动作的最终边界

核心原则：
1. 未获同意，永不写入

---

## 8. 设计原则
1. 默认静默，不默认接管聊天
2. 普通用户先获得预置模板，而不是被迫学复杂编排
3. 资源池是为了复用和组合，不是为了让用户从零配置一切
4. 阶段和 agent 必须解耦
5. recorder 与 watchdog 必须分离
6. 写边界必须独立于编排能力
7. 用户退出微操，但保留治理权

---

## 9. 当前开放问题
1. `COMPLEX` 是否默认总带 recorder
2. `HUGE` 的阶段模板是固定系统包，还是任务自定义优先
3. `MAX` 第一阶段是否先限制在单个 `managedTaskRef` 内，不做子 managed task
4. `AcceptancePolicy` 是否在 `COMPLEX` 也强制存在
5. 普通用户看到的是“阶段”，还是更自然的“工作模式/任务流程”
