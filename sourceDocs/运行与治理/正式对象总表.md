# 正式对象总表

更新时间：2026-03-11

---

## 1. 使用方式
这份表只回答四件事：
1. 这个名字是什么
2. 它代表什么语义
3. 它属于哪一层
4. 它的唯一 owner 在哪里

固定说明：
1. `aggregate` 表示有独立 contract owner 的正式聚合根
2. `成员对象` 表示值对象或子实体，不单独拥有平级 owner
3. `pointer/ref` 表示稳定引用，不单独要求独立合同
4. `projection` 表示用户可见或宿主兼容投影，不是 authoritative state
5. `transport carrier` 表示宿主或 adapter 传输载体，不进入 Loom 领域主链
6. `标量枚举/默认值` 表示固定值域或顶层默认值
7. `正式操作` 表示正式命令或系统动作，不是领域聚合根
8. `禁用别名/禁止口径` 表示历史错误名词、错误字段或禁止写法，不得当正式对象直接实现

## 6. 禁用别名/禁止口径
| 名称 | 当前状态 | 为什么不能直接实现 | 正确写法 |
| --- | --- | --- | --- |
| `EscalationPolicy` | 禁用泛化正式对象名 | 当前正式 owner 只有 `WipPolicy / ReworkPolicy / AcceptancePolicy` 各自的 `...escalation_policy`，泛化名会制造假 owner | 顶层显式写 policy 来源 |
| `child_session_key` | 禁止继续当正式字段名 | 它是 session 型宿主的 compat alias，把它抬成领域字段会把 OpenClaw 传输语义污染进宿主无关合同 | 使用 `host_child_execution_ref` |
| `child_run_id` | 禁止继续当正式字段名 | 它是旧 transport 兼容字段，不是宿主无关正式引用名 | 使用 `host_child_run_ref` |
| `review_summary_excerpt` | 禁止字段别名 | 顶层链路图发明了不存在的 payload 字段名 | 回到正式字段 `proof_of_work_excerpt.review_summary` |
| `risk_profile` | 禁止平行对象族写法 | 第一版风险 owner 已收回到 `RiskAssessment`，单独再立对象会和 assessment 抢真相源 | 使用 `RiskAssessment.overall_risk_band / risk_dimensions` |

## 7. durable 判定规则
固定规则：
1. 只要某个对象会影响 `ControlAction` 是否有效、`review/result` 是否成立、或 run 是否允许继续执行，它就必须有 durable owner。
2. 只负责解释、推荐或背景说明的内容，不得在 live docs 里伪装成平行 durable owner。
3. 不允许同一治理判断同时存在 authoritative owner 和一套实现层自由拼装的临时 owner。

---

## 8. 当前使用建议
写代码时，优先遵守下面这条规则：
1. 先查对象在这张表里的层级
2. 再去对应 owner 文档读字段和边界
3. 如果名字落在“禁用别名/禁止口径”里，就不要直接拿去实现或再建新 contract
