# 运行时基线

更新时间：2026-03-11

---

## 1. 文档定位
这份文档只记录 Loom 运行时的固定基线，也就是未来代码实现必须遵守的 target runtime baseline。

固定边界：
1. 这不是 current implementation inventory
2. 这不是旧系统遗留盘点
3. 它只回答 Loom 运行时哪些事实必须成立

---

## 2. authoritative truth
### 2.1 唯一真相源
Loom 的唯一治理真相源固定为：
1. `runtime/loom/`

### 2.2 真相源职责
它至少承接：
1. task aggregate
2. append-only task events
3. outbound delivery authoritative state
4. review/result artifacts 的结构化引用

### 2.3 compatibility projection
宿主兼容投影只允许作为 compatibility projection。

它不再承接：
1. 产品规范
2. 任务状态真相源
3. future development 的 owner 角色

---

## 3. 运行时主对象
### 3.1 输入对象
1. `CurrentTurnEnvelope`
   - 当前宿主输入的权威入站对象
2. `HostSemanticBundle`
   - 宿主语义层给出的综合结构化判断包
3. `SemanticDecisionEnvelope`
   - adapter 归一化后的 bounded judgment
4. `HostCapabilitySnapshot`
   - 宿主当前现实能力快照

### 3.2 任务对象
1. `managedTaskRef`
   - 单个受管任务 owner
2. `activeManagedTaskRef`
   - 当前唯一 active task 指针
3. `TaskScopeSnapshot`
   - 当前任务范围与工程上下文快照
4. `PhasePlan`
   - 当前任务实际采用的阶段序列
5. `workflowStage`
   - `candidate / execute / review / result / closed`
6. `AgentBinding`
   - 当前 run 实际生效的 agent roster
7. `SpecBundle`
   - 正式执行锚点文档组
8. `IsolatedTaskRun`
   - 执行期隔离 run

### 3.3 治理对象
1. `PendingDecisionWindow`
2. `pendingUserDecision`
3. `pendingBoundaryConfirmation`
4. `ExecutionAuthorization`
5. `RiskAssessment`
6. `AcceptancePolicy`

## 4. 固定运行时约束
### 4.1 语义边界
1. Loom 不做自然语言语义推断。
2. 宿主先产出 `HostSemanticBundle`。
3. adapter 只做 schema 校验和归一化。

### 4.2 任务边界
1. `managedTaskRef` 是任务 owner。
2. `host_session_id` 只是聊天容器。
3. active task 不冻结主聊天。

### 4.3 阶段边界
1. 第一条真实主链也必须显式包含：
   - `review`
2. `candidate` 到 `execute` 必须先经过 start card 决策。
3. `clarify / source_collection / synthesis / deliver`
   - 都属于 `PhasePlan / StagePackageId`
   - 不是 `workflowStage` 扩展枚举

### 4.4 delivery 边界
1. authoritative state/event/outbox 先提交。
2. adapter 再读取并投递。
3. v0 采用 `at-least-once`，不假装 `exactly-once`。

### 4.5 bridge 边界
1. 代码 spike 固定使用 `LocalHttpBridge`。
2. bridge 固定：
   - `loopback-only`
   - `shared bridge secret`
3. bootstrap ticket 与 session credential 分离。

### 4.6 会话并行边界
1. `taskConversationPolicy=chat_open`
   - 当前只是顶层产品默认值
2. 它不是 formal runtime object
3. v0 不要求把它持久化进 `runtime/loom/`

### 4.7 durable 治理边界
1. 会影响 `ControlAction` 有效性、`review/result` 成立性、或 run 是否允许继续执行的对象，必须进入 `runtime/loom/` 或其正式结构化引用面。
2. 解释类背景、研究类判断、用户可见 projection 不得伪装成第二套 durable owner。
3. 第一版不引入独立 `AcceptanceLedger / ResponsibilityGraph / AttentionPolicy` 主链。

## 5. 当前推荐实现基线
1. `Rust + TypeScript adapter + bridge-first`
2. `SQLite + append-only task events + projections`
3. `loom-openclaw` 作为第一个宿主 adapter
4. 第一条真实 spike：
   - `coding_pack / COMPLEX`
   - `net + recorder + 1 worker`
