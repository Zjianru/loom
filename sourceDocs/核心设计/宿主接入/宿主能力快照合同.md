# 宿主能力快照合同

状态：contract draft  
定位：`HostCapabilitySnapshot` 正式规格  
更新时间：2026-03-10

---

## 1. 目的
`HostCapabilitySnapshot` 定义宿主在某个时刻向 Loom 同步的正式能力快照。

它回答：
1. 宿主当前有哪些 agent、model、tool、render 和 worker control 能力
2. 哪些能力可以直接影响 `AgentBinding / ExecutionAuthorization / CapabilityDriftAssessment`
3. 宿主能力同步如何成为正式输入对象，而不是只停留在 OpenClaw 接口稿里
4. `HostWorkerControlCapabilities` 在整个治理链里到底属于哪一层

更深层上，它解决的是：

**把“宿主当前现实能力”从某个单一宿主的接口描述，升级成 Loom 可复用的正式输入合同。**

---

## 2. 设计边界
### 2.1 这不是 `AgentCapabilityProfile`
边界：
1. `AgentCapabilityProfile`
   - 是 Loom 内核里的抽象画像
2. `HostCapabilitySnapshot`
   - 是宿主当前现实可提供的能力事实

### 2.2 这不是 `AgentBinding`
边界：
1. `HostCapabilitySnapshot`
   - 回答宿主能给什么
2. `AgentBinding`
   - 回答这次 run 真正选了谁上场

### 2.3 这不是 `ExecutionAuthorization`
边界：
1. `HostCapabilitySnapshot`
   - 是现实能力输入
2. `ExecutionAuthorization`
   - 是在这些现实能力上收敛出的当前执行租约

### 2.4 这不是 `CapabilityDriftAssessment`
边界：
1. `HostCapabilitySnapshot`
   - 是新旧对比前的原始事实
2. `CapabilityDriftAssessment`
   - 是 Loom 对这些事实差异的治理结论

---

## 3. v0 固定规则
1. adapter 启动时至少同步一次 `HostCapabilitySnapshot`
2. agent/model/tool/worker control 发生实质变化时，必须重新同步
3. `HostCapabilitySnapshot` 是宿主无关正式对象，不得写成 OpenClaw 私有字段表
4. `HostWorkerControlCapabilities` 必须是快照的一部分，不能散落在 pause/cancel 文档里单独持有 owner
5. 能力增强不自动扩大当前 `ExecutionAuthorization`
6. 能力缩减必须进入 `CapabilityDriftAssessment`

## 4. 正式对象定义
```rust
pub struct HostCapabilitySnapshot {
    pub snapshot_id: HostCapabilitySnapshotId,
    pub host_kind: HostKind,
    pub available_agents: Vec<HostAgentCapability>,
    pub available_models: Vec<HostModelCapability>,
    pub available_tools: Vec<HostToolCapability>,
    pub render_capabilities: HostRenderCapabilities,
    pub background_task_support: bool,
    pub async_notice_support: bool,
    pub worker_control_capabilities: HostWorkerControlCapabilities,
    pub observed_at: Timestamp,
}

pub struct HostAgentCapability {
    pub host_agent_ref: HostAgentRef,
    pub display_name: String,
    pub available: bool,
}

pub struct HostModelCapability {
    pub host_model_ref: HostModelRef,
    pub provider: String,
    pub available: bool,
}

pub struct HostToolCapability {
    pub tool_name: String,
    pub available: bool,
}

pub struct HostRenderCapabilities {
    pub supports_text_render: bool,
    pub supports_inline_actions: bool,
    pub supports_message_suppression: bool,
}

pub struct HostWorkerControlCapabilities {
    pub supports_pause: bool,
    pub supports_resume: bool,
    pub supports_cancel: bool,
    pub supports_soft_interrupt: bool,
    pub supports_hard_interrupt: bool,
}
```

---

## 5. 关键字段分别代表什么
1. `snapshot_id`
   - 当前能力快照的稳定 id
2. `host_kind`
   - 这份快照来自哪个宿主类型
3. `available_agents / available_models / available_tools`
   - 宿主当前现实可见且可映射的 agent、model、tool 清单
4. `render_capabilities`
   - 宿主当前文本渲染与动作呈现现实
5. `background_task_support / async_notice_support`
   - 宿主是否支持后台任务与异步通知
6. `worker_control_capabilities`
   - 正在运行的 worker 到底能否被 pause、resume、cancel 或 interrupt

---

## 6. 和其它对象的关系
### 6.1 和 `AgentBinding`
固定关系：
1. `Harness` 先读取最新 `HostCapabilitySnapshot`
2. 再结合 `Pack / AgentPool / ExecutionAuthorization`
   收敛出 `AgentBinding`

### 6.2 和 `ExecutionAuthorization`
固定关系：
1. `HostCapabilitySnapshot`
   - 是 lease 收敛的现实约束
2. 当前宿主真实有能力
   - 不等于当前 run 自动放行

### 6.3 和 `CapabilityDriftAssessment`
固定关系：
1. 每次新快照进入后
2. Loom 都可以形成正式 drift assessment
3. assessment 再决定 keep、narrow、reissue、pause 或 require user decision

### 6.4 和 `HostWorkerControlCapabilities`
固定关系：
1. 它是 `HostCapabilitySnapshot` 的子对象
2. pause/resume/cancel 文档只能引用它
3. 不再单独持有字段 owner

---

## 7. 我的建议
1. `HostCapabilitySnapshot` 应作为宿主无关正式输入合同存在，不再只停留在 OpenClaw 接口文档里
2. `HostWorkerControlCapabilities` 继续由这份合同持有 owner，其它文档只引用
3. adapter 真实实现时，任何 capability sync 都应优先写成这份对象，而不是先写宿主私有 projection
