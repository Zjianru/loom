## 1. 现在这套文档在描述什么
`sourceDocs/` 现在统一描述 **Loom 套件** 的未来开发基线。  
这不是设计草稿堆，也不是讨论纪要堆。

它当前按“产品入口 -> 正式规范 -> 实施验收 -> 运行治理 -> 核心设计 -> 研究补充”的顺序组织，只服务一件事：

**围绕 Loom、Harness、approval-gate、watchdog、loom-openclaw 与 `runtime/loom/` 来完成未来开发。**

更细的对象合同、接口形状、示例和实现切线，统一放在 [核心设计目录说明.md](../核心设计/目录说明.md)。

## 4. 当前固定约束
1. 顶层文档现在以 Loom 为唯一未来开发主线。
2. `Harness` 是核心治理运行时名称。
3. `loom-openclaw` 是首个宿主 adapter。
4. `runtime/loom/` 是唯一 authoritative truth root。
5. kernel 不做自然语言语义推断，只消费宿主结构化判断。
6. `managedTaskClass` 的正式值为：
   - `COMPLEX`
   - `HUGE`
   - `MAX`
7. `coding_pack / research_pack` 是当前唯一写透的 pack preset。
8. `SpecBundle / ProofOfWorkBundle / IsolatedTaskRun` 是 Loom 通用能力，不是某个单独 pack 的技巧。
9. 风险收敛到 `RiskAssessment` 对象族，不另开平行记录对象族。

---

## 5. 使用原则
1. 需要方向和边界时，先读顶层规范链。
2. 需要对象字段、协议和实现切面时，转到 `核心设计/`。
3. 需要开工时，先看 `产品实施总方案` 与测试文档。
4. 正式产品验收只认 `OpenClaw WebUI` 单入口。
5. 顶层规范只围绕 Loom 套件本身展开。
6. 不要再从研究/讨论文档中寻找当前对象 owner 或字段真相。
